# Systemic Circulation Modeling and Feedback Control

This project merges the original circulation exercises into one reusable MATLAB codebase. It keeps the same three scientific scenarios while moving shared parameters, simulation logic, hemorrhage inputs, and plotting into a common `shared/` layer.

## Project Structure

- `baseline_hemorrhage/`: open-loop systemic circulation during hemorrhage
- `methods_validation/`: Euler, eigen-decomposition, and matrix-exponential validation for the same circulation model
- `feedback_control/`: controlled versus uncontrolled circulation during hemorrhage
- `shared/`: reusable model, simulation, input, and plotting functions

## Main Entry Points

- `baseline_hemorrhage/run_baseline_hemorrhage.m`
- `methods_validation/run_methods_validation.m`
- `feedback_control/run_feedback_control.m`
- `verify_circulation_project.m`

## Refactor Notes

- Shared constants are centralized in `shared/circulation_params.m`.
- The hemorrhage input is generated by `shared/make_hemorrhage_profile.m`.
- Open-loop and controlled simulations are separated from plotting and return reusable result structs.
- The runner scripts save standardized figure outputs into each scenario folder's `figures/` directory.
- The methods-validation scenario now validates the same flow law used by the hemorrhage and feedback scenarios:
  - `q = Kr * (Pra - 1.82)`
- Because that flow law introduces a constant offset, the validation problem is treated as an affine linear system around equilibrium rather than as a homogeneous linear system.

## Verification

Run `verify_circulation_project.m` to:
- execute the three scenario entry points
- regenerate the standardized figures
- confirm output files are produced
- check for empty arrays, NaNs, Infs, and basic dimension mismatches

## Scope

This refactor improves code quality and project structure. It does not intentionally introduce new physiology or claim new numerical findings beyond the existing scenarios.
